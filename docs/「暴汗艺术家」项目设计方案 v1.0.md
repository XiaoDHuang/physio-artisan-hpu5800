# 「暴汗艺术家」项目设计方案 v1.0

**——基于思维链提示词与 LangGraph 多智能体协同的闭环健康决策系统**

------

## 一、项目核心定位

本项目面向健康管理场景，旨在解决传统健身应用“静态建议、缺乏因果推理”的局限。核心设计思路是：**以分层提示词工程驱动多智能体协同推理**，而非单纯依赖 API 调用堆叠。

具体而言，系统在 LangGraph 框架下，将健康决策任务拆解为多个专家智能体节点。每个智能体持有一套独立的系统提示词，并受思维链（Chain of Thought）约束，按照既定步骤对穿戴设备采集的生理数据进行递进式推理。各智能体通过共享状态协同工作，最终输出逻辑可追溯、安全可控的个性化干预方案。

系统产出的多模态体检报告（趋势图表与语音简报），目的是将智能体的推理过程和决策依据以可视化、可收听的方式呈现，使用户能够理解“AI教练为何给出这样的建议”，而非仅仅看到一个结果数字。

------

## 二、案例与数据设计（演示载体）

为避免答辩演示流于空谈，项目设计了严格的**“对照组（恶化路径） vs 实验组（向好路径）”**单人案例。

### 2.1 虚拟用户画像

- **姓名**：小明
- **实际年龄（Chronological Age, CA\*C\**A\*）**：30岁
- **健身目标**：科学减脂，提升综合体能，优化**身体年龄（Body Age）**
- **数据基座**：华为 Watch GT 6 Pro（支持睡眠、心率、HRV、运动强度等时序数据 JSON 导出）

### 2.2 我们获取并分析的数据

| 数据类别    | 具体指标                                                  | 数据来源                    | 生物力学/生理学意义                    |
| :---------- | :-------------------------------------------------------- | :-------------------------- | :------------------------------------- |
| 睡眠监测    | 总睡眠时长、深睡比例、REM比例、苏醒次数、睡眠评分         | 手表 JSON 导出              | 评估中枢神经恢复与肌肉修复窗口         |
| 心血管/恢复 | 晨间静息心率（RHR）、心率变异性（HRV）、心率恢复力（HRR） | 手表 JSON 导出              | HRV 越低越疲劳；HRR 反映副交感神经活性 |
| 运动负荷    | 昨日训练时长、平均心率、自评 RPE（1-10）                  | 手表运动记录 + 用户手动输入 | 计算 TRIMP，评估累计生理压力           |
| 饮食记录    | 三餐食材名称、估算分量                                    | 用户文字描述或拍照          | 调用 USDA API 配平热量及蛋白质         |
| 身体测量    | 体重、体脂率                                              | 用户体脂秤数据手动录入      | 用于校准身体年龄预测模型               |

### 2.3 两套核心数据设计（放任恶化 vs 智能体拯救）

#### ① 对照组（无智能体干预）——放任恶化路径

模拟小明在连续高强度训练、睡眠不足的情况下，放任身体恶化 7 天的数据：

- **睡眠评分**：70分 → 65分 → 58分（进入警戒区，深睡比例 <15%）
- **HRV**：45ms → 32ms（显著低于历史基线）
- **静息心率**：62bpm → 68bpm（持续升高，提示过度训练）
- **HRR**：18bpm → 12bpm（心脏恢复极慢，<18bpm 健康标准）
- **饮食与训练**：蛋白质仅 0.8g/kg，无视疲劳继续进行大重量深蹲（RPE 9）
- **身体年龄（BA）**：35.0岁 → 预测 7 天后升至 35.5岁

#### ② 实验组（经多智能体 3 天干预后）——积极恢复路径

展示小明采纳智能体“降低运动负荷、补充抗炎营养、强制恢复”建议 3 天后的逆转数据：

- **睡眠评分**：回升至 80分（深睡比例恢复至 20%）
- **HRV**：回升至 45ms（中枢疲劳消除）
- **静息心率**：降回 63bpm（心肺负荷解除）
- **HRR**：恢复至 20bpm（副交感神经成功激活）
- **饮食与训练**：蛋白质达 1.8g/kg，增加 Omega-3 抗炎脂肪酸，训练降载为核心激活与低强度有氧（RPE 4）
- **身体年龄（BA）**：由 35.0岁 安全降至 34.6岁

## 四、大脑中枢：LangGraph 与分层提示词设计（技术制高点）

系统彻底摒弃单一超级提示词，将决策逻辑拆分为由 LangGraph 统一调度、各司其职的专家智能体网格。

**有状态共享机制（State）**：定义统一的共享字典 `AgentState`，通过 SQLite（`SqliteSaver`）自动保存每一步的运行快照，支持答辩现场随时回滚，模拟宕机数据恢复。

**条件边（Conditional Edge）**：由生理评估员输出的疲劳等级（`fatigue_level`），动态决定是否触发运动降载和营养干预。

### 4.2 专家提示词微调与调教（Prompt Engineering 精华）

#### ① 生理评估智能体 (State Evaluator)

**核心逻辑**：基于穿戴指标计算生理准备度指数（Readiness Score, RS*RS*），并输出疲劳等级。

text

```
[角色] 你是顶级运动生理学专家，专精于通过时序传感器特征判断人体的中枢疲劳状况。
[输入] 昨日手表数据JSON：睡眠评分、HRV、静息心率（RHR）、心率恢复力（HRR）、TRIMP。

# 思考链 (Chain of Thought) 调教约束：
你必须严格按照以下三步进行递进式推理，不得直接输出结论：
Step 1: 计算 Readiness Score (RS)。利用非线性加权：
        RS = 0.5 * (今日HRV / HRV基准) + 0.3 * (睡眠评分 / 100) + 0.2 * (基线RHR / 今日RHR)
        并将其折算为百分制数字。
Step 2: 评估系统疲劳红相。检查以下四项，每命中一项疲劳计数+1：
        - HRV 低于基线 80%
        - 睡眠评分 < 70
        - HRR < 18 bpm
        - 静息心率连续3天升高
        命中0项 → 疲劳度“低”；命中1-2项 → 疲劳度“中”；命中3项及以上 → 疲劳度“高”
Step 3: 确定最终疲劳度等级，并生成一句大白话解读（Insight），用通俗语言解释疲劳主因。

[输出格式] 必须输出严格的 JSON 字符串，禁止输出任何多余的 Markdown 标记：
{ "rs": 62, "fatigue": "high", "rhr_status": "elevated", "insight": "小明，你昨日的静息心率偏高且睡眠严重不足，中枢神经恢复差。今日必须进行运动降载。" }
```

#### ② 运动教练智能体 (Exercise Coach)

**核心逻辑**：检测到中枢疲劳时，自主调用 wger API 检索低负荷拉伸和核心代偿激活方案。

text

```
[角色] 你是拥有 NSCA-CSCS 认证的教练，精通动态降载（Deload）原则。
[输入] 生理评估员输出的 JSON + 用户的历史训练负荷。

# 思考链 (Chain of Thought) 调教约束：
Step 1: 判定今日恢复状态。如果 RS < 65 且 HRR < 18 bpm，判断今天属于“CNS高疲劳日”。
Step 2: 实施降载决策。为了保护腰椎和关节，今天必须取消大重量深蹲，限制 TRIMP 负荷。
Step 3: 自主检索降载方案。你必须使用 `wger_exercise_search` 工具，检索关键词 "core stretch" 或 "mobility"。
Step 4: 撰写新旧计划对比。用一句话因果推导，解释为何用核心拉伸代替杠铃负重。

[回退规则] 如果 API 调用失败或超时，你必须基于专业知识给出替代动作建议，并在输出 JSON 中标记 "source": "fallback_knowledge"。

[可用工具] `wger_exercise_search(query="stretch", limit=3)`
[输出格式] JSON：
{ "training_type": "recovery", "exercises": [...], "reason": "昨晚睡眠严重欠债，中枢控制力下降。为防止高强度深蹲时发生代偿，今日换成核心稳定与拉伸动作。", "source": "wger_api" }
```

#### ③ 膳食规划智能体 (Nutritional Planner)

**核心逻辑**：基于运动消耗与油脂代偿，调用 USDA 数据库配平高蛋白促恢复膳食。

text

```
[角色] 你是获得国际注册运动营养师（ISENC）认证的营养专家。
[输入] 昨日总能量消耗、今日减脂目标、昨日饮食摄入。

# 思考链 (Chain of Thought) 调教约束：
Step 1: 动态能差计算。以今日总消耗（TDEE + 今日运动消耗）为基准，减脂期维持温和能差 -300 kcal。
Step 2: 宏量营养配平。抗阻恢复期，蛋白质摄入目标定为 1.8g * 体重(kg)。
Step 3: 油脂代偿代扣。检测到“沙拉”、“油炸”、“酱料”等描述时，必须对检索出的食材基准热量乘以油脂代偿系数 λ_sauce = 1.30，防止热量低估。
Step 4: 检索权威库。你必须且只能使用 `usda_food_search` 检索食材，严禁凭空捏造任何食物的微量元素数字。

[回退规则] 如果 API 调用失败或超时，你必须基于专业知识给出替代食材建议，并在输出 JSON 中标记 "source": "fallback_knowledge"。

[可用工具] `usda_food_search(keyword="salmon")`
[输出格式] JSON：
{ "total_calories": 1850, "protein_target_g": 144, "diet_suggestion": "晚餐建议补充150g煎三文鱼配糙米饭，已调用USDA数据库配平今日蛋白质缺口。", "source": "usda_api" }
```

#### ④ 安全审计智能体 (Guardrail Auditor)——稳健防御红线

**核心逻辑**：扮演独立的“LLM-as-a-Judge”角色，无死角审计运动和饮食计划，拦截医疗药物和高危建议。

text

```
[角色] 你是独立的安全合规官（Guardrail Auditor），拥有最终一票否决权。

[检查红线列表]：
- 绝不允许出现任何处方药或临床消炎药物名称（如“双氯芬酸钠”、“布洛芬”、“利尿剂”、“睾酮”、“生长激素”、“塞来昔布”、“依托考昔”、“对乙酰氨基酚”等）
- 绝不允许出现违禁兴奋剂（如“氮泵”高剂量、“克伦特罗”）
- 绝不允许出现激素类药物（如“胰岛素”、“甲状腺素”）
- 绝不允许给出病理诊断建议（如“你可能是腰椎间盘突出”、“你应该是肌腱炎”），凡涉及剧烈或持续疼痛，必须建议就医
- 绝对拦截高危极限饮食建议（如“24小时断食”）和危险训练动作

# 思考链 (Chain of Thought) 调教约束：
Step 1: 扫描上游生成的全文字段，逐词与红线词库比对。
Step 2: 如果命中任何一条红线，判定为 UNSAFE，立即触发安全熔断，无条件用安全拦截模板覆盖整段话术。
Step 3: 如果无任何风险，判定为 SAFE，加盖 [安全通行] 戳，原文直接放行。

[安全熔断模板]：
"安全拦截警告 [Guardrail Active]：检测到提问涉及临床医学建议/药物指导。AI智能体无法提供处方药方案。如果训练后腰部剧烈刺痛，请立即暂停所有抗阻运动，并寻求专业医生进行诊疗。"

[输出格式] JSON：
{ "status": "passed/blocked", "reason": "...", "final_text": "..." }
```

#### ⑤ 报告生成智能体 (Report Generator)——多模态枢纽

**核心逻辑**：汇总所有专家思考结果，提炼出 200 字极具人情味的总结，并组装成前端渲染所需的多模态结构体。

text

```
[角色] 你是xx专业的健康管家。
[输入] 上述所有节点的决策 JSON、小明的历史生理数据。
[任务]：
1. 汇总当前关键指标（RS分数、睡眠质量、今日降载训练类型、高蛋白食谱配平数据）。
2. 计算当前指标与 7 天前的变化量（Δ），用来在体检报告中标记“↑改善”或“↓恶化”。
3. 撰写一段 200 字以内的多模态语音播报文本（Vocal Narrative），语气需专业、温暖、令人信赖，专门用于 edge-tts 神经网络女声合成。结尾需附上安全声明：“本报告由AI生成，不能替代专业医疗诊断。如身体持续不适，请及时就医。”

[输出格式] 严格的 JSON，传给前端 Streamlit 直接渲染：
{ "visual_metrics": { "rs": 62, "body_fat": 24.0, "body_age": 35.0 }, "vocal_narrative": "早上好，小明。昨天的数据显示你处于重度疲劳状态，静息心率偏高。我今天已经帮你把大重量深蹲调整为核心舒缓训练，晚餐配平了三文鱼，记得带上耳机跟练哦。最后提醒，本报告由AI生成，不能替代专业医疗诊断。如身体持续不适，请及时就医。" }
```

## 六、实验组 vs 对照组对比表（答辩核心数据）

通过两组数据的量化对比，我们不吹嘘一周练出腹肌，而是向评委证明智能体能在 3 天内将身体拉回积极恢复轨道的科学决策质量。

| 生理学核心指标   | 对照组（放任恶化，7天预测值）  | 实验组（智能体干预 7 天后）             | 数据逆转幅度        | 算法/数据物理支撑               |
| :--------------- | :----------------------------- | :-------------------------------------- | :------------------ | :------------------------------ |
| 睡眠综合评分     | 58分（Pay Attention 危险区）   | 80分（Good 优质睡眠）                   | ↑ 38%               | 华为 GT 6 Pro 睡眠深度监测      |

## 七、演示流程脚本（提示词与多智能体协同展示）

1. **项目概括**

   - **Nutritional Planner** 节点高亮：今日训练量下调，重新计算 TDEE，调用 USDA API 检索 Salmon 配平蛋白质缺口。
